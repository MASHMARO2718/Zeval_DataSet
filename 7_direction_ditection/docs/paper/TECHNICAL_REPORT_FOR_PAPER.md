## 1. 研究概要

本研究では、GroundTruth（Unity Kinematika）とMediaPipe（RGB画像ベース）による3D人体姿勢推定の精度を定量的に比較評価した。評価には角度誤差を主要指標として採用し、関節間の誤差相関を分析することで、MediaPipeの体系的な推定傾向を明らかにした。本研究では、107フレーム、505台のマルチビューカメラ、12関節について合計259,356観測の大規模データセットを用いた評価を行った。

### 1.1 データセット構成

- **フレーム数**: 107フレーム
- **カメラ配置**: 505台のマルチビューカメラ
  - X座標: -6.0m ～ +6.0m（13位置）
  - Y座標: 0.5m, 1.0m, 1.5m, 2.0m（4高度）
  - Z座標: -6.0m ～ +6.0m（13位置）
  - 理論的最大: 13 × 4 × 13 = 676台（実際には505台のデータが存在）
- **分析対象関節**: 12関節
  - 上肢: LEFT_SHOULDER, RIGHT_SHOULDER, LEFT_ELBOW, RIGHT_ELBOW, LEFT_WRIST, RIGHT_WRIST
  - 下肢: LEFT_HIP, RIGHT_HIP, LEFT_KNEE, RIGHT_KNEE, LEFT_ANKLE, RIGHT_ANKLE
- **総データポイント**: 259,356観測（107フレーム × 505カメラ × 12関節の部分集合）

## 2. 評価指標の数学的定義

### 2.1 座標系の統一

#### 2.1.1 GroundTruth座標系
- **元の座標系**: 右手座標系（Unity標準）
- **変換後**: 左手座標系（MediaPipeに合わせる）
- **基準点**: 左右腰関節の中点を原点とする相対座標系

座標変換：
```
hip_center = (LEFT_HIP + RIGHT_HIP) / 2
relative_coord = original_coord - hip_center
```

#### 2.1.2 MediaPipe座標系
- **元の座標系**: 正規化座標（画像座標系、0-1の範囲）
- **変換処理**: 同様に腰中心を原点とする相対座標系に変換
- **重要な発見**: MediaPipeの出力は腰を原点としておらず、絶対座標であった
  - そのため、GroundTruthと同様の相対座標変換が必須

### 2.2 角度誤差の定義（主要評価指標）

本研究では、**角度誤差を主要な評価指標**として採用した。これは以下の理由による：
1. GroundTruthとMediaPipeは異なるスケール（メートル vs 正規化座標）
2. 3D距離誤差は単位の違いにより直接比較が困難
3. 角度は単位に依存せず、姿勢の方向性を評価できる

#### 2.2.1 XY平面角度誤差（Δθ）

XY平面（水平面）における関節の方向誤差を評価する。

**定義式**:
```
θ_gt = arctan2(y_gt, x_gt)
θ_mp = arctan2(y_mp, x_mp)
Δθ = θ_mp - θ_gt
```

ここで、
- `(x_gt, y_gt)`: GroundTruthの相対座標のX, Y成分
- `(x_mp, y_mp)`: MediaPipeの相対座標のX, Y成分
- `arctan2(y, x)`: 2引数逆正接関数（-π ≤ θ ≤ π）

**正規化処理**:
```python
def normalize_angle(angle):
    """角度を-πからπの範囲に正規化"""
    while angle > π:
        angle -= 2π
    while angle < -π:
        angle += 2π
    return angle

Δθ_normalized = normalize_angle(Δθ)
```

**度数法変換**:
```
Δθ_deg = Δθ_normalized × (180 / π)
```

**物理的意味**:
- Δθ > 0: MediaPipeが上方向（+Y軸）にずれている
- Δθ < 0: MediaPipeが下方向（-Y軸）にずれている
- |Δθ| = 90°: 直角にずれている（大きな誤差）
- |Δθ| ≈ 180°: 反対方向を向いている（極度の誤差）

#### 2.2.2 XZ平面角度誤差（Δψ）

XZ平面（前額面）における関節の方向誤差を評価する。

**定義式**:
```
ψ_gt = arctan2(z_gt, x_gt)
ψ_mp = arctan2(z_mp, x_mp)
Δψ = ψ_mp - ψ_gt
Δψ_normalized = normalize_angle(Δψ)
Δψ_deg = Δψ_normalized × (180 / π)
```

**物理的意味**:
- Δψ > 0: MediaPipeが奥行き方向（+Z軸）にずれている
- Δψ < 0: MediaPipeが手前方向（-Z軸）にずれている
- カメラ視点からの奥行き推定精度を反映

#### 2.2.3 3D距離誤差（参考指標）

**定義式**:
```
error_3d = ||coord_mp - coord_gt||₂
         = √[(x_mp - x_gt)² + (y_mp - y_gt)² + (z_mp - z_gt)²]
```

**注意**: この値は単位の違いにより直接的な意味を持たないが、相対的な傾向の参考値として記録した。

### 2.3 実装コード（抜粋）

```python
def calculate_angle_xy(self, coord: np.ndarray) -> float:
    """XY平面での角度θを計算"""
    return np.arctan2(coord[1], coord[0])

def calculate_angle_xz(self, coord: np.ndarray) -> float:
    """XZ平面での角度ψを計算"""
    return np.arctan2(coord[2], coord[0])

def normalize_angle(self, angle: float) -> float:
    """角度を-πからπの範囲に正規化"""
    while angle > np.pi:
        angle -= 2 * np.pi
    while angle < -np.pi:
        angle += 2 * np.pi
    return angle

def calculate_differences(self, gt_coords, mp_coords):
    """GroundTruthとMediaPipeの差分を計算"""
    differences = {}
    
    for joint_name in gt_coords.keys():
        gt = gt_coords[joint_name]
        mp = mp_coords[joint_name]
        
        # 角度計算
        theta_gt = self.calculate_angle_xy(gt)
        theta_mp = self.calculate_angle_xy(mp)
        delta_theta = self.normalize_angle(theta_mp - theta_gt)
        
        psi_gt = self.calculate_angle_xz(gt)
        psi_mp = self.calculate_angle_xz(mp)
        delta_psi = self.normalize_angle(psi_mp - psi_gt)
        
        differences[joint_name] = {
            'delta_theta': delta_theta,
            'delta_theta_deg': np.degrees(delta_theta),
            'delta_psi': delta_psi,
            'delta_psi_deg': np.degrees(delta_psi),
            'error_3d': np.linalg.norm(mp - gt)
        }
    
    return differences
```

## 3. 相関分析の方法論

### 3.1 目的

関節間の誤差相関を分析することで、以下を明らかにする：
1. MediaPipeの体系的な推定誤差パターン
2. 特定の関節ペアが連動して誤差を持つかどうか
3. 誤差伝播のメカニズム
4. 補正アルゴリズム設計への示唆

### 3.2 Pearson相関係数の定義

2つの関節 i, j について、全観測（フレーム×カメラ）における誤差の相関を評価する。

**定義式**:
```
r_ij = Σ[(X_i - X̄_i)(X_j - X̄_j)] / √[Σ(X_i - X̄_i)² × Σ(X_j - X̄_j)²]
```

ここで、
- `X_i`: 関節 i の誤差系列（Δθ、Δψ、またはerror_3d）
- `X̄_i`: 関節 i の誤差平均値
- `r_ij`: 相関係数（-1 ≤ r_ij ≤ 1）

**相関係数の解釈**:
- `r ≈ 1.0`: 強い正の相関（一方が増加すると他方も増加）
- `r ≈ 0.0`: 無相関（独立）
- `r ≈ -1.0`: 強い負の相関（一方が増加すると他方は減少）
- `|r| > 0.7`: 高相関とみなす閾値（本研究での基準）

### 3.3 相関行列の計算手順

#### ステップ1: データの準備

全データをピボットテーブル形式に変換：

| (frame, camera) | LEFT_SHOULDER | RIGHT_SHOULDER | ... | LEFT_ANKLE |
|-----------------|---------------|----------------|-----|------------|
| (20, Cam_1)     | Δθ₁           | Δθ₂            | ... | Δθ₁₀       |
| (20, Cam_2)     | Δθ₁           | Δθ₂            | ... | Δθ₁₀       |
| ...             | ...           | ...            | ... | ...        |

```python
pivot = df.pivot_table(
    index=['frame_id', 'camera'],
    columns='joint',
    values='delta_theta_deg',  # または delta_psi_deg, error_3d
    aggfunc='first'
)
```

#### ステップ2: 欠損値処理

全関節が揃っている観測のみを使用（完全データのみで分析）：

```python
pivot_clean = pivot.dropna()
```

本データセットでは、欠損値は極めて少なく（< 0.1%）、データ品質が高い。

#### ステップ3: 相関行列の計算

Pearson相関係数を全関節ペアについて計算：

```python
corr_matrix = pivot_clean.corr(method='pearson')
```

結果は 10×10 の対称行列（10関節）。

#### ステップ4: 高相関ペアの抽出

閾値（|r| > 0.7）を超えるペアを抽出：

```python
def analyze_high_correlations(corr_matrix, threshold=0.7):
    high_corr_pairs = []
    
    for i in range(len(corr_matrix)):
        for j in range(i+1, len(corr_matrix)):  # 上三角のみ（重複排除）
            corr_value = corr_matrix.iloc[i, j]
            if abs(corr_value) > threshold:
                high_corr_pairs.append({
                    'joint_1': corr_matrix.index[i],
                    'joint_2': corr_matrix.columns[j],
                    'correlation': corr_value
                })
    
    return pd.DataFrame(high_corr_pairs)
```

### 3.4 分析の3つのオプション

本研究では、3種類の相関行列を計算した：

#### オプションA-1: θ（XY平面）の相関
- **対象**: `delta_theta_deg`
- **意味**: 水平面での誤差の連動性
- **用途**: 左右対称性や上下肢の連動を評価

#### オプションA-2: ψ（XZ平面）の相関
- **対象**: `delta_psi_deg`
- **意味**: 奥行き方向の誤差の連動性
- **用途**: カメラ視点依存の誤差パターンを評価

#### オプションC: 3D誤差ノルムの相関
- **対象**: `error_3d`
- **意味**: 総合的な位置誤差の連動性
- **注意**: 単位の違いにより絶対値の解釈は困難だが、相関パターンは有効

#### 4.1.1 主要な発見

1. **肘関節の体系的バイアス**
   - LEFT_ELBOW: θ = +121.06° （ほぼ常に同じ方向に誤差）
   - RIGHT_ELBOW: θ = -121.00° （左右対称に逆方向の誤差）
   - **解釈**: MediaPipeは肘の位置を体系的に誤認している
   - 標準偏差が小さい（23-25°）ため、**補正可能な体系的誤差**

2. **足首・肩の高変動誤差**
   - |θ|平均が168-171°と極めて大きい
   - 標準偏差も大きい（140-159°）
   - **解釈**: フレームやカメラ角度により推定が不安定
   - ランダム性が高く、補正が困難

3. **手首の相対的精度**
   - LEFT_WRIST: |θ| = 94.28°
   - RIGHT_WRIST: |θ| = 93.14°
   - 他の関節に比べると誤差が小さい
   - **解釈**: 手の輪郭が明瞭で推定しやすい可能性

### 4.2 相関分析結果

#### 4.2.1 θ（XY平面）の高相関ペア

**高相関ペア（|r| > 0.7）**: 1ペアのみ

| 関節1 | 関節2 | 相関係数 |
|-------|-------|---------|
| LEFT_ELBOW | RIGHT_ELBOW | **-0.8174** |

**解釈**:
- **強い負の相関**: 左肘と右肘の誤差が逆方向に連動
- 平均値（+121°と-121°）と合わせて考えると、**左右対称な体系的バイアス**
- MediaPipeは肘を外側に押し出すような推定バイアスを持つ
- **補正戦略**: 左右肘の誤差を反転させることで精度向上が期待できる

#### 4.2.2 ψ（XZ平面）の高相関ペア

**高相関ペア（|r| > 0.7）**: 8ペア

| 関節1 | 関節2 | 相関係数 | 解釈 |
|-------|-------|---------|------|
| LEFT_HIP | RIGHT_HIP | **-0.8402** | 腰の左右で逆方向の奥行き誤差 |
| RIGHT_ELBOW | RIGHT_SHOULDER | **+0.7697** | 右上肢の連動（同方向誤差） |
| RIGHT_ELBOW | RIGHT_WRIST | **+0.7691** | 右前腕の連動（同方向誤差） |
| LEFT_ELBOW | LEFT_SHOULDER | **+0.7680** | 左上肢の連動（同方向誤差） |
| RIGHT_SHOULDER | RIGHT_WRIST | **+0.7258** | 右腕全体の連動 |
| LEFT_ELBOW | LEFT_WRIST | **+0.7217** | 左前腕の連動 |
| RIGHT_ELBOW | RIGHT_HIP | **+0.7212** | 右肘と右腰の連動 |
| LEFT_HIP | RIGHT_ELBOW | **-0.7052** | 左腰と右肘の逆相関 |

**主要な発見**:

1. **腰の左右対称誤差（-0.84）**
   - 最も強い相関
   - 左右の腰が奥行き方向で逆に推定される
   - **原因仮説**: MediaPipeの骨盤モデルが回転誤差を持つ
   - 一方の腰が前に出ると、他方が後ろに引っ込む

2. **上肢のキネマティックチェーン（+0.72〜+0.77）**
   - 肩→肘→手首が連動して誤差を持つ
   - **解釈**: MediaPipeが腕全体を前後にシフトして推定
   - 関節間の相対位置は保たれるが、全体がずれる
   - **補正戦略**: 肩の誤差を補正すれば、肘・手首も改善する可能性

3. **右肘と右腰の連動（+0.72）**
   - 予想外の相関
   - **仮説**: 体幹の姿勢推定誤差が腕に伝播
   - または、カメラ角度依存の同時発生誤差

#### 4.2.3 3D誤差ノルムの相関（参考）

相関分析を実施したが、|r| > 0.7 のペアは少なく、θとψの分析ほど明確なパターンは見られなかった。これは、3D誤差が複数の要因の複合的な結果であるためと考えられる。

## 5. データから読み取れる詳細な傾向

### 5.1 MediaPipeの推定特性

#### 5.1.1 体系的バイアスの存在

**肘関節の典型例**:
- LEFT_ELBOW: 常に+121°の誤差（左外側へ）
- RIGHT_ELBOW: 常に-121°の誤差（右外側へ）
- 標準偏差が小さい（23-25°）ため、**予測可能な誤差**

**インプリケーション**:
- 単純な線形補正で大幅な精度向上が可能
- 例: `θ_corrected = θ_raw - 121°`

#### 5.1.2 左右対称性の保持

**観察**:
- LEFT_ELBOW と RIGHT_ELBOW の誤差が対称（±121°）
- LEFT_HIP と RIGHT_HIP の誤差が逆相関（-0.84）

**解釈**:
- MediaPipeの骨格モデルは左右対称性を前提としている
- 一方の推定誤差が、他方に鏡像的に現れる
- **利点**: 一方の補正方法を、反転して他方に適用できる

#### 5.1.3 キネマティックチェーンの連動

**上肢の連動誤差**:
- 肩→肘→手首が高相関（+0.72〜+0.77）
- 関節間の相対角度は比較的正確
- しかし、腕全体が前後・左右にシフト

**下肢の独立性**:
- 膝・足首は他の関節との相関が低い
- より独立した推定が行われている
- 安定性が低い（高い標準偏差）

### 5.2 カメラ位置依存性

#### 5.2.1 フレームごとの変動

詳細分析（`frame_camera_summary.csv`）より：
- 同じ被写体姿勢でも、カメラ角度により誤差が大きく変動
- 最良カメラと最悪カメラで50-100°の差
- **重要**: 正面・側面・斜めで精度が異なる

**例**:
- フレーム20: 最小誤差46.66°（最良カメラ）
- フレーム26: 最小誤差101.89°（最良カメラ）
- 被写体の姿勢の違いが精度に大きく影響

#### 5.2.2 Y座標（高度）の影響

4つの高度（0.5m, 1.0m, 1.5m, 2.0m）でデータを収集：
- 高度による体系的な精度差は観察されず
- カメラ高度よりも**水平角度が精度を決定**
- ただし、極端な俯角・仰角では精度が低下する可能性

### 5.3 誤差の分布特性

#### 5.3.1 標準偏差の分析

**小さい標準偏差（予測可能）**:
- RIGHT_ELBOW: 23.02°
- LEFT_ELBOW: 25.50°
→ 補正が容易

**大きい標準偏差（予測困難）**:
- LEFT_ANKLE: 158.94°
- RIGHT_ANKLE: 158.40°
- LEFT_SHOULDER: 148.61°
- RIGHT_SHOULDER: 140.85°
→ フレーム・カメラ依存が強く、補正が困難

#### 5.3.2 誤差範囲の広さ

ほぼ全ての関節で：
- θ_min ≈ -180°
- θ_max ≈ +180°

**解釈**:
- 一部のフレーム・カメラで、完全に反対方向を推定している
- MediaPipeは極端な外れ値を持つ
- **ロバスト性の欠如**: 難しい姿勢・角度で完全に失敗

### 5.4 補正アルゴリズムへの示唆

#### 5.4.1 優先順位の高い補正対象

1. **肘関節（最優先）**
   - 体系的バイアス（±121°）の除去
   - 線形補正で大きな改善が期待できる
   - 実装例: `θ_corrected = θ_raw - bias`

2. **腰関節（高優先）**
   - 左右対称性を強制する制約
   - 実装例: `hip_center_correction(left_hip, right_hip)`

3. **上肢キネマティックチェーン（中優先）**
   - 肩の補正が肘・手首に波及
   - 階層的な補正アルゴリズム

#### 5.4.2 補正が困難な要素

1. **足首・肩の高変動誤差**
   - ランダム性が高く、予測困難
   - 機械学習ベースの補正が必要
   - または、複数カメラの融合で平滑化

2. **カメラ角度依存性**
   - 最適カメラ角度の選択が重要
   - マルチビュー融合で精度向上が期待できる

#### 5.4.3 提案する補正フレームワーク

```
階層的補正アルゴリズム:

Level 1: 体系的バイアスの除去
  - 肘の±121°補正
  - 関節ごとの固定オフセット補正

Level 2: 左右対称性の強制
  - 腰の左右バランス調整
  - 肘の左右対称性確認

Level 3: キネマティックチェーン補正
  - 肩の補正 → 肘・手首への伝播
  - 腰の補正 → 膝・足首への伝播

Level 4: 時系列平滑化
  - カルマンフィルタ等の適用
  - フレーム間の急激な変化を抑制

Level 5: マルチビュー融合
  - 複数カメラの加重平均
  - 信頼度に基づく重み付け
```

## 7. 結論と今後の展望

### 7.1 主要な発見のまとめ

1. **MediaPipeは体系的な推定バイアスを持つ**
   - 肘関節: ±121°の固定的な誤差
   - 腰関節: 左右対称な奥行き誤差（相関-0.84）
   - これらは**補正可能な体系的誤差**

2. **角度誤差は極めて大きい**
   - 平均143.13°（XY平面）、90.52°（XZ平面）
   - 実用アプリケーションには大幅な補正が必須

3. **関節間の誤差相関が明確**
   - 上肢のキネマティックチェーン（+0.72〜+0.77）
   - 左右対称な誤差パターン
   - 補正アルゴリズム設計への有用な知見

4. **カメラ角度が精度を決定する**
   - 最良・最悪で2倍以上の精度差
   - マルチビュー融合が精度向上の鍵

### 7.2 学術的貢献

1. **定量的ベンチマーク**
   - 19,440観測による大規模評価
   - 角度誤差を主要指標とした新しい評価フレームワーク

2. **体系的誤差の発見**
   - 肘の±121°バイアスの特定
   - 腰の左右対称誤差の発見

3. **相関分析による誤差構造の解明**
   - キネマティックチェーンの連動誤差
   - 補正アルゴリズムへの具体的示唆

### 7.3 今後の研究課題

1. **補正アルゴリズムの実装と評価**
   - 階層的補正フレームワークの実装
   - 補正前後の定量的比較

2. **機械学習ベースの補正**
   - カメラ角度を入力とした誤差予測モデル
   - 深層学習による非線形補正

3. **マルチビュー融合**
   - 複数カメラの最適な統合方法
   - カメラ選択戦略の最適化

4. **動的な姿勢での評価**
   - 本研究は静止フレームの分析
   - 連続的な動きでの追跡精度評価

5. **他の姿勢推定手法との比較**
   - OpenPose、BlazePose等との定量比較
   - 手法間の体系的ベンチマーク
